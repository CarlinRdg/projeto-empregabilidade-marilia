# 2. Cronograma Macro

| Semana | Atividade |
|----------|----------|
| Semana 1 | Planejamento do projeto e definição dos requisitos |
| Semana 2 | Coleta dos dados do Novo CAGED e IBGE |
| Semana 3 | Tratamento e preparação dos dados |
| Semana 4 | Modelagem e implementação do PostgreSQL |
| Semana 5 | Desenvolvimento do pipeline ETL em Python |

| Semana 8 | Relatório final, GitHub e entrega |

# 3. Matriz de Riscos

| Risco | Impacto | Probabilidade | Mitigação |
|---------|---------|---------|---------|
| Indisponibilidade dos dados públicos | Alto | Baixa | Utilizar versões históricas já baixadas |
| Dados incompletos ou inconsistentes | Alto | Média | Realizar validações durante o ETL |
| Dificuldade na integração entre ferramentas | Médio | Média | Desenvolver e testar cada etapa separadamente |
| Atraso no cronograma | Médio | Média | Definir entregas semanais |
| Problemas de versionamento dos arquivos | Médio | Baixa | Utilizar Git e GitHub durante todo o projeto |
